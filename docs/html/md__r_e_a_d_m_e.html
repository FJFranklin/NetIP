NetIP - TCP/IP networking multiple devices over serial

For project website and documentation, see here.

Overview

The aim of NetIP is to be able to link multiple devices into a network using serial connections, so that any device in - or connected to - the network can talk to any other. This is achieved using Serial Line Internet Protocol (SLIP) between devices, and forwarding packets transparently at intermediate nodes. UDP is supported, and also a subset of TCP/IP. (There is no support for fragmentation and reassembly.)

The principal target of NetIP is embedded devices, so the program code is small and memory requirements small, with no dynamic allocation of memory that would lead to fragmentation. The two 'architectures' currently supported are 'Unix' (tested on Linux and Mac OS X) and 'Arduino' (tested on the Uno, Due and Teensy 3.5).
